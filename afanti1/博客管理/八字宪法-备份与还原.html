八字宪法-备份与还原

2010-7-28

备份（backup）

俗话说，狡兔三窟。俗话还说，人老了奸，兔子老了滑。看来在俗话里，可爱的兔子并不那么可爱，可是特实惠。长期捣腾数据的人跟兔子一样，都知道定期做数据后备的重要性，而且知道做备份的最好结果就是永远都用不着去动用备份的数据。在万维网上做备份到底有没有必要，有多大用，怎么说呢，咱们这有吃过苦的，问问他们就知道了。

备份做起来很简单。走到你想留底儿的那篇文章，然后用  File -> Save page as … 保留那篇文章到你的硬盘上（文章和所有的评论都会存起来的）。注意，在用 Save page as 时，保留文件的类型要选用 “Web Page,  complete” 。重复这个步骤，直到把所有文章的备份都做完。

有一个特别值得注意的事是：如果选用“Web Page,  complete”作为保留文件的类型，浏览器会将所有的文字、图片下载并将备份文件存放在磁盘上，但不会将多媒体文件（录像，音乐等）下载备份。所以如果文章中有音像链接的话，而且这些音像文件是你自己的文件，你需要把它们做另外的备份，如果只是从网上找到的他人的作品，你需要另外保存那个链接。（但愿浏览器在这方面以后会有所改进）。如果选用“All files ” 作为保留文件的类型，浏览器会将所有的文字（正文和留言）下载并将备份文件存放在磁盘上，而其他内容将会以链接的形式备份。因此你可以根据具体情况选择不同的备份文件类型。（IE 和 Firefox 对保留文件类型的定义是不同的，若用IE，最好先试试）

如果你很多文章，而且从来没做过备份，可以考虑使用下载工具将你博客下的所有文章一次性地下载到硬盘上作为备份。（在 download.com 上用 download manager 查找，可以找到许多免费的下载软件）。另外如果你使用 FireFox 浏览器的话，还可以考虑使用它的一些插件（plug-in，add-on）最为备份工具。使用这些工具的好处是可以一次性地、成批地从万维博客下载多篇文章。当然也需要花些时间来学习使用这类软件。

万维博主欢乐颂在他的博客上介绍了一种这类软件的使用方法，言简意赅，非常清楚。这里是那篇文章的链接，可以参考：鸣谢网管，兼谈博文的“一键”备份

还原 （restore）

由于种种技术原因，在没有万维网管的支持下，现在不可能把丢失的文章和留言百分之百地照原样还原，但可以做到相当的接近。如果不小心把一篇文章从博客中删掉了，可以用备份文件重新发表这篇文章，但不可能以其他网友的名义在这篇文章后面重新留言。如果这些留言很重要，而且想保留这些留言的话，可以把这些留言作为文章的一部分，附在文章的后面并保留原来的格式，这样看起来十分接近原来的样子。包括留言人的链接，留言的时间等都可以原封不动地保留下来。做还原时，需要有些使用 HTML 编辑软件的知识。具体做法如下：

用浏览器浏览备份文件，然后选择（highlight）正文和留言（从标题到结尾）

拷贝所选的文字然后将其粘贴到 HTML 编辑软件的编辑窗口中去 （Ctrl c, Ctrl v）

在 HTML 编辑器的设计窗口里，删除标题行和一些无效的链接（修改浏览次数到三百万 ）。最后点击“code”转到 HTML 代码窗口。

拷贝 HTML 代码的 body 部分，并将其粘贴到发表日志的代码窗口。

如果需要的话，在 HTML 编辑器里做调整，然后重复这个过程直到满意为止。丢了东西会很不愉快，但能把它找回来，还是会使人相当高兴的。
