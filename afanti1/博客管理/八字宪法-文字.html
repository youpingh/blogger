八字宪法-文字

2010-7-28

对我来说，处理好文字是第一件重要的事了。有下面几个要注意的地方。

中文输入法

对用拼音的人来说，选一个顺手的的拼音输入软件对有效地码字很有帮助。现在有好几种不同的软件可以挑选：谷歌拼音 ，搜狗拼音，微软拼音等。我自己用谷歌拼音（谢谢多思的推荐），觉得不错。本山大哥在春晚给搜狗做过广告，应该也不错（更重要的是，米笑博主觉得这个方法不错）。当然了，实在不行了，还有微软的拼音输入法，应该不是全世界最差的。

文本编辑软件（Text/HTML Editor）

除非是只写一两句评论，所有的文字最好先用文本编辑软件写好后再上传，所以选一个好用的文本编辑软件也相当重要。市面上和网上有很多种文本编辑软件可选，微软的 Word 大概是最流行的一个了。其实Word 并非最佳选择，因为Word 在你的文字背后添加了许多不可见的控制字符来控制文字的大小，颜色，字体和格式等。在上传文字时时，这些控制字符也会被上传，有时它们会造成一些意想不到的显示问题，比如说乱码，出格，文字丢失等等。（在后面的“格”字篇里，会谈到一些防止格式混乱的具体办法）。我个人认为最好是用一个 HTML 编辑软件来编辑文字。理由很简单，因为网上几乎所有的文字都是用 HTML 控制显示的，如果你用 HTML 编辑软件写东西，因格式转换而造成的问题就会减少到最少。

网上有许多免费下载和使用的 HTML 编辑器，在 www.download.com 上用 HTML EDITOR 查找可以找到一大把。注意选用那些可以做到“所见即所得”（WYSIWYG What-You-See-Is-What-You-Get ) 的那种。现在这类软件已经做得很好用了，如果有使用 Word 的经验，掌握使用个 HTML 编辑软件应该不困难。另外如果对 HTML 不熟悉并想学习一下的话，有个网上学校办得不错，对初学者和专业人员都有用。http://www.w3schools.com/html/default.asp

简繁体设置

万维对博客正文的简繁体显示的处理做得很到家，但对评论文字的简繁体显示处理有漏洞。基于多数博主使用简体汉字，我在浏览万维网时，特别是在写评论时，将浏览器的字符编码 （Character Encoding）设置为 Chinese Simplified GB2312。这样在写评论时，就不会在别人的文章下留下不同字符编码 （比如说繁体编码 Big5）的评论了。留下不同字符编码评论的直接后果就是你的评论在人家的文章中无法显示，于是你的评论就白写了。 

     上传文章

文章上传质量的好坏直接影响到文章的可读性，如果读者仅仅是因为上传质量的问题而跳过一篇精彩的文章，那对读者和作者都是一大损失（我码的字是否精彩当然得由读者来定，我就不在这假谦虚了）。如果使用一个功能很强的文本编辑软件写东西，比如说微软的 Word，那上传文章时最要紧的事就是把那个软件加在文章中所有多余的控制字符去掉，以避免那些字符带来的显示问题。有两种简单的办法去掉那些控制字符。第一，发表日志时用万维提供的“粘贴为无格式文本”的方式：（这个办法是从转悠那抄来的）

点击“T”符号，然后粘贴文字。这种方式会把文本编辑软件加在你文章中的控制字符全部去掉，因此也就将所有的显示隐患去掉了。缺点是你在编辑文本时所做的所有努力也都被去掉了，比如说字体，颜色等。在“确定”后你还得重新编辑格式。

第二个办法是用万维提供的“删除格式”的功能。用这种办法时，可将文字直接粘贴到发表日志的文字窗口内，然后用“删除格式”将大部分不可见的格式控制字符去掉。（这个办法是从多思那学来的）

粘贴后点击“删除格式”符号。这种办法的好处是有选择地去掉那些控制字符，有些有用的还留在文中，比如说段落空格。但如果你在文中使用了 Word 的“编号(number)”或“项目符号 (bullets)”等编辑手段，这些控制符号仍有可能造成显示的混乱。

有时 IE 和 Firefox 对文章中的文字会有不同的显示处理，其结果是在一个浏览器下看是好的，可在另一个下看却是乱码。所以发表日志后用两个浏览器检查一下很必要，特别是如果使用了一些 Word 的特殊编辑手段后。

如果使用 HTML 编辑软件的话，上面所有的这些麻烦就全都没有了。只要把HTML 编码粘贴上去就好了。而且不用再做任何编辑（字体，大小，颜色等等）。在发表日志时，点击“代码”，然后粘贴。

我在编辑一些比较复杂的日志时，就用 HTML 编辑器，比如说我的 文章目录，效果就很理想。
